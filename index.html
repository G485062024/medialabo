「Webコンテンツ作成」 の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得する。「Webコンテンツ作成」ではHTML,CSS,JavaScriptなどを使ってWebを制作できるようにする。

プロジェクト課題について

このプロジェクト課題については、WWW のサーバへ検索し、検索結果を表示するページの作成に取り組みました。NHKの番組表を検索して、表示するページを制作しました。ワードが一致していないと検索できないことや、どのようなものがあるのか想像しずらいということからプルダウンのようなものを用意して選んでもらうようにしました。また、チャンネルの検索ボタンを表示しないようにしてわかりやすいようにしました。

プロジェクト課題で作成したページ: project.html

数当てゲーム

乱数で 1〜10の値を1つ決めて，「答え」を決め、ユーザは答えを予想して，数を最大で3回まで入力できるゲームを作りました。 ユーザの入力が外れた場合は，入力よりも答えが大きいか小さいか表示して、3回以内に答えの数を当てればユーザの勝ち，そうでなければ負けとなるように制作しました。ページには，テキストボックスとボタンが設置されていて、ユーザがテキストボックスに予想した数を入力してボタンを押すと判定がされるようになっています。 課題3-1では4をユーザが入力したとして制作した。hantei()が実行された際に、回答が4回以上の場合・正解の場合・間違いの場合などについての条件分岐が起こるようにプログラムを制作した。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

Web ページを提供する仕組みのことを WWW といいインターネット上には膨大な数の Web ページが存在する。
また、Web ページを構成する「部品」は大きくHTML,CSS,JavaScript,画像・音声・動画の四つに分けられる。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSはページを装飾するもので、要素の見た目を変更するものになります。そのためHTMLが最初に与えられて次にその装飾にCSSを記述するという順番になります。CSSをHTMLに適用する方法は三つあり、外部スタイイルシート・内部スタイルシート・インラインスタイル・がありますが今回は外部スタイルシートを使って記述しています。

演習の成果: 演習2-3: Webブラウザ説明ページの装飾

第3回 JavaScript の基本

JavaScriptはWeb ページ上で複雑な機能を実現するプログラミング言語でWeb ページを開いた直後や，ボタンを押したときに JavaScript プログラムが実行されます。動更新やアニメーションなど動きのあるページを実現したい時に使われるプログラムです。

演習の成果: 演習3-6:オブジェクトの参照

第4回 DOM と JavaScript

今回は「ボタンを押したらページが変化する」処理を JavaScript プログラムとして記述する方法を学びました。また、DOM（Document Object Model）は Web コンテンツを表現するデータ構造です、DOM を使用することによって、JavaScript などのプログラムによって、Web コンテンツの内容を読み取ったり，コンテンツに変更を加えたりする処理が書けるようになりました。

演習の成果: 演習4-10:足し算ページ

第5回 イベント処理

今回はDOMの高度な機能についての学習を行いました。前回は DOM の機能のうち，検索とテキストの参照，変更，input要素の値取得に絞って学習しましたが、今回は要素の新規作成・要素の編集（要素の属性を追加したり変更したり）・要素の削除などの機能についての学習でした。

演習の成果: 演習5-7:工学部の学科名一覧を表示する

工夫点，苦心点，強調点，感想

今回の演習では、WWW と HTML の基礎から始まり、CSS による装飾、JavaScript による動的挙動、そして DOM 操作とイベント処理を段階的に学びました。第 1 回では HTML で文書の意味と構造を意識し、見出し・段落・リストなどのタグを使用しました。第 2 回では外部スタイルシートを選び、配色や余白を整えて一貫したデザインを目指しました。第 3 回では変数宣言・条件分岐・関数定義など JavaScript の基本構文を習得し、ページ読み込み時やクリック時にスクリプトが動く仕組みを確かめました。console.log を使ったデバッグで処理の流れを追いながら、エラー発生時の原因特定力も鍛えられました。第 4・5 回では DOM を介して要素を生成・削除・置換し、フォームの値を即座に画面へ反映させる動きを実装しました。addEventListener によるイベント登録で、クリックや入力イベントを柔軟にハンドリングできるようになり、ページの一部だけを更新する仕組みを理解しました。こうして構造・見た目・挙動を整理して開発する手順を身に付けられたと感じます。 プロジェクト課題では、グルメ情報や世界の天気は検索キーが一つしかないのに対して、NHK の番組表だけ検索キーが二つあることに苦労しました。ジャンルとチャンネルを区別するために、それぞれのプログラムに NHK 総合か E テレかを判断する文を追加し、ユーザーが両方の条件を直感的に選択できるよう UI をプルダウン方式に変更しました。検索キーが二つあるため、両方選択が必須となる仕様を明確に伝え、不要なチャンネル切替ボタンを削除して操作性を向上させました。最初に、NHK の番組表を表示する際に、表のような形で区切って表示したかったのですが、WWW で検索したものを表示するとなったときに、そのままだと使えなくなってしまい、ul と li 要素に変更してしまったので、NHK の番組表をもっとわかりやすく表示できるように作ることができたらもっと良かったなと考えています。また、検索した結果として表示するもので文字数が多すぎたりするものがあると、微妙なところで改行されてしまい見ずらいものになってしまうことがあるのでそういったことにも対応できるようなページ作りができるように勉強していきたいと思います。実際に多くの学生がweb作りを仕事にするだろうといった話もあったので将来に向けて選択肢を増やせるようにしていきたいと思います。
